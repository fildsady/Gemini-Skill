# Global Coding Rules

นี่คือไฟล์ตั้งค่าสกิลแบบ Global (ครอบคลุมทุกโปรเจกต์) คุณสามารถแก้ไขและเพิ่มกฎเกณฑ์ที่ต้องการให้ AI ปฏิบัติตามลงในไฟล์นี้ได้เลยครับ

## สไตล์การเขียนโค้ดที่ต้องการ (Coding Standards)
- เขียนโค้ดที่สะอาด (Clean Code) มีการใส่คอมเมนต์อธิบายโค้ดส่วนที่ซับซ้อนอย่างเหมาะสม
- รักษารูปแบบการเขียนโค้ดและโครงสร้างเดิมของโปรเจกต์เมื่อเข้าไปแก้ไข
- เน้นการแบ่งส่วนโค้ดเป็นโมดูล (Modular Code) เพื่อความง่ายต่อการพัฒนาและทดสอบ

## ความชอบในการทำงาน (Workflow Preferences)
- ก่อนการแก้ไขไฟล์สำคัญ ให้อธิบายแนวทางการแก้ไขแบบสรุปสั้น ๆ ก่อนเสมอ
- หลีกเลี่ยงการลบส่วนโค้ดหรือคอมเมนต์เก่าที่ไม่เกี่ยวข้องกับการเปลี่ยนแปลง เว้นแต่จะสั่งโดยตรง
